# SP-1 Data-Model Extension Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the credit-core Prisma schema (MySQL) additively so it can hold every field a filled CLEVER Mikromoliya workbook contains that import + scoring + payment-schedule + document-generation need.

**Architecture:** Add 4 enums and 13 new models hanging off the existing `CreditCase` (the workflow dossier stays untouched), plus field additions to `Borrower`/`Collateral`/`CollateralOwner`. Reconcile pre-existing migration drift into a clean baseline first, then create one `sp1_data_model` migration. Prove the model with a round-trip fixture that maps the real TADJIYEV sample deal into every relation.

**Tech Stack:** Prisma 6 (`prisma-client-js`), MySQL 8.4 (Docker, port 3307, `lower_case_table_names=1`), NestJS, ts-node, TypeScript.

## Global Constraints

- **Additive only.** Do not remove or rename existing fields/models. The `CreditCase` workflow/SLA/chat/`Document` machinery is untouched.
- **Money** columns are `Decimal @db.Decimal(18, 2)`; **rate/fraction** columns are `Decimal @db.Decimal(6, 4)` (e.g. 0.55, 1.05).
- **Amount/term "in words" (прописью) are NOT stored** — generated at render time by SP-2. Do not add `*Words` fields beyond the ones already in the schema.
- **Prisma engine network workaround (mandatory on this machine — flaky network hangs engine downloads).** Before EVERY `prisma` command, export:
  ```bash
  export CHECKPOINT_DISABLE=1
  export PRISMA_QUERY_ENGINE_LIBRARY="C:/Users/Administrator/Desktop/credit-core/node_modules/@prisma/engines/query_engine-windows.dll.node"
  export PRISMA_SCHEMA_ENGINE_BINARY="C:/Users/Administrator/Desktop/credit-core/node_modules/@prisma/engines/schema-engine-windows.exe"
  ```
  (See project memory `local-dev-setup`. These point Prisma at the already-downloaded local engines.)
- **Dev DB:** Docker MySQL on host port **3307**, `DATABASE_URL="mysql://root:root@localhost:3307/credit_core"` in `apps/backend/.env`. The container must keep `lower_case_table_names=1` (already set in `docker-compose.dev.yml`).
- Run all `prisma`/`ts-node` commands from `apps/backend` (so `.env` is picked up).
- Reference for field meanings/sample values: `docs/analysis/2026-06-29-mko-workbook-analysis.md` §2 (data dictionary). Design source of truth: `docs/superpowers/specs/2026-06-29-sp1-data-model-design.md`.
